# 研究分析层执行说明

## 1. 本阶段目的

研究分析层的目的不是直接给战略结论，而是为后续“具体机会点洞察”提供证据基础。

本阶段需要完成四类分析：

1. 行业趋势分析；
2. 消费者需求 / 心智分析；
3. 竞品格局分析；
4. 企业自身分析。

本阶段不得提前输出最终机会点、战略大单品候选、战略承接方案。

## 2. 输入资料

优先读取：

- 客户资料；
- 品牌手册；
- 产品手册；
- 官网资料；
- 行业报告；
- 政策文件；
- 消费者调研资料；
- 竞品官网 / 旗舰店 / 公开资料；
- 用户提供的竞品名单。

不得把以下内容作为研究依据：

- MEMORY.md；
- agent_memory；
- 项目执行记录；
- 确认点；
- 用户反馈；
- Skill 规则文件；
- AGENTS.md。

如果读取 MEMORY.md，只能用于判断当前执行阶段，不得进入研究依据库。

## 3. 行业趋势分析

需要回答：

1. 行业处于什么阶段；
2. 行业规模、增长、政策、技术、渠道、消费变化有哪些关键趋势；
3. 这些趋势对客户所在赛道意味着什么。

输出要求：

- 不堆数据；
- 每个数据必须服务一个判断；
- 没有可靠数据时必须标注“资料不足，需补充验证”；
- 不得编造市场规模、增速、份额等数据。

## 4. 消费者需求 / 心智分析

需要回答：

1. 消费者在这个品类中真正关心什么；
2. 消费者有哪些未被满足的痛点；
3. 消费者如何表达自己的需求；
4. 哪些词是消费者能感知、能表达、能传播的。

输出要求：

- 优先使用消费者原话、评论、调研资料；
- 区分企业语言和消费者语言；
- 不得直接把“专业、可信赖、有效、健康、高端、功能场景”等企业侧词汇当作消费者机会点。

## 5. 竞品格局分析

每个核心竞品默认分析 1-2 页对应的信息量。

必填维度：

1. 品牌定位是什么；
2. 优势渠道是什么；
3. 核心人群是谁；
4. 优势产品是什么。

可补充维度：

- 核心卖点；
- 价格带；
- 内容传播方式；
- 销售表现；
- 消费者认知；
- 代表性营销动作。

输出要求：

- 先逐个竞品拆解；
- 再做横向共性对比；
- 最后输出竞争格局阶段结论；
- 不得在竞品分析阶段直接输出“客户应该占据什么机会点”；
- 不得设置“客户机会点”列。

## 6. 企业自身分析

需要回答：

1. 企业有什么资产；
2. 企业有什么短板；
3. 哪些资产能被消费者感知；
4. 哪些资产只是企业内部能力，不能直接作为消费者选择理由；
5. 哪些产品、渠道、研发、供应链、品牌资产可能支撑后续机会点。

输出要求：

- 区分“企业优势”和“消费者选择理由”；
- 不得把企业资质直接等同于品牌机会点；
- 不得直接在本阶段指定战略大单品。

## 7. 本阶段输出格式

研究分析层输出时，建议按照以下结构：

一、行业趋势分析

- 关键发现
- 支撑资料
- 阶段判断

二、消费者需求 / 心智分析

- 关键发现
- 消费者语言
- 阶段判断

三、竞品格局分析

- 逐个竞品拆解
- 横向共性对比
- 阶段判断

四、企业自身分析

- 资产扫描
- 问题判断
- 阶段判断

五、进入确认点 2 的准备

- 可以进入机会点洞察的证据
- 仍需补充验证的资料

## 8. 本阶段禁止事项

禁止：

- 提前输出最终机会点；
- 提前输出战略大单品候选；
- 提前输出战略承接方案；
- 把企业优势直接写成机会点；
- 把泛词写成机会点；
- 把 MEMORY.md、agent_memory、确认点、用户反馈作为研究依据；
- 编造行业数据、销售数据、市场份额；
- 在竞品分析阶段直接给客户机会空位；
- 用长篇报告代替结构化研究结论。
