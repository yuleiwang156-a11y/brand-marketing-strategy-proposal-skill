# 逐页 PPT 内容脚本生成执行说明

## 1. 本阶段目的

本阶段的目的，是在确认点 3 已通过后，根据客户可见一级目录和内部分析结构，逐部分生成 Word 版 PPT 页面内容脚本。

本阶段不是重新做研究，不是重新推导机会点，也不是自由发挥写报告。

## 2. 启动条件

只有在以下条件满足后，才能进入逐页生成阶段：

1. 确认点 1 已通过；
2. 研究分析层已完成；
3. 确认点 2 已通过；
4. 确认点 3 已通过；
5. 客户可见一级目录已确认；
6. 内部分析结构已确认。

如果确认点 3 未通过，不得直接生成逐页脚本。

## 3. 分批输出规则

逐页脚本必须按客户可见一级目录分批输出。

不得默认每 10 页输出一批。

每一批开始前必须先输出：

```markdown
本批生成内容：第 X 部分「章节名」

本章任务：
1. ……
2. ……
3. ……

本章不回答：
1. ……
2. ……
3. ……

本章预计页面：
第 X 页：……
第 X+1 页：……
```

然后再开始逐页生成。

每一批结束后必须暂停，等待用户确认。

用户确认后，才能继续下一部分。

## 4. 页面标准格式

每一页必须使用以下格式：

```markdown
第 X 页：页面标题

页面类型：
目录页 / 章节页 / 结论页 / 分析页 / 数据页 / 对比页 / 表格页 / 金句页 / 反问页 / 路径页 / 总结页

页面正文：
……

资料依据：
……
```

不得缺少页码、页面标题、页面类型、页面正文、资料依据。

## 5. 页面标题规则

页面标题必须是观点型标题。

标题应该直接表达本页判断，而不是普通名词。

好的标题：

- 当前问题不是产品太多，而是缺少清晰的增长主线
- 消费者真正能记住的是具体状态，而不是企业资质
- 竞品普遍在讲专业，但没有形成可传播的消费词

不好的标题：

- 客户情况
- 产品分析
- 竞品介绍
- 行业分析
- 机会点说明

## 6. 页面正文规则

页面正文必须像 PPT 页面脚本，不得像长报告。

要求：

1. 每页只讲一个核心观点；
2. 正文优先使用 2-4 条结构化要点；
3. 每页正文原则上控制在 150-250 字；
4. 不写长段小作文；
5. 不在一页里塞入多个复杂判断；
6. 分析页要有阶段判断；
7. 数据页要有“这说明什么”；
8. 表格页必须服务一个明确结论。

## 7. 章节边界规则

不同章节只回答本章该回答的问题。

项目概况：
只讲项目背景、客户诉求、问题缘起、研究路径。
不得提前输出机会点、大单品候选、战略承接结论。

问题诊断：
只讲表层问题、零散问题归因、表面问题与根本问题。
不得输出具体机会点和产品承接。

环境分析：
只讲行业、消费者、竞品、企业自身。
不得输出客户最终机会点和大单品候选。

机会洞察：
正式输出四维交叉分析、具体机会点、机会点具体性检查和最终机会点清单。

战略承接：
在机会点确认后，输出产品承接、渠道承接、传播承接和推进节奏。

## 8. 资料依据规则

资料依据优先引用：

1. 客户资料；
2. 品牌手册；
3. 产品手册；
4. 官网 / 官方资料；
5. 行业报告；
6. 政策文件；
7. 消费者调研资料；
8. 竞品官网 / 旗舰店 / 公开资料；
9. 研究分析层中的资料来源。

不得把以下内容作为客户版页面主要资料依据：

- 确认点 1；
- 确认点 2；
- 确认点 3；
- 用户反馈；
- Skill 规则；
- AGENTS.md；
- MEMORY.md；
- agent_memory；
- 项目执行记录。

如果需要引用内部过程判断，可写：

“依据前序研究分析。”

但不得在客户版资料依据中写“依据确认点 2 用户反馈”。

## 9. 信息释放顺序

逐页稿必须按以下顺序释放信息：

客户现状与问题
→ 行业环境
→ 消费者证据
→ 竞品证据
→ 企业自身证据
→ 机会点
→ 战略承接
→ 推进节奏

不得在前文提前泄露后文章节的结论。

## 10. 页数规则

完整建议书原则上控制在 50-60 页。

少于 50 页，说明论证可能不足。
超过 60 页，说明需要压缩合并。

但不得为了页数牺牲核心论证。

## 11. 暂停确认规则

每完成一个客户可见一级目录对应部分，必须暂停。

暂停时输出：

```markdown
本部分到此暂停。
请确认是否通过。
通过后可回复：第 X 部分通过，继续第 X+1 部分。
```

不得自动连续生成多个一级目录。

## 12. Word 生成规则

最终生成 Word 文档时：

1. 优先使用 python-docx 直接生成 .docx；
2. 不默认调用 LibreOffice；
3. 不默认调用 soffice；
4. 不打开本地 Office / LibreOffice 图形界面；
5. 如果无法直接生成 docx，应先保留 Markdown 或文本版完整内容，再提示用户处理。

## 13. 本阶段禁止事项

禁止：

- 确认点 3 未通过就生成逐页脚本；
- 默认按每 10 页一批输出；
- 跨章节提前释放后文结论；
- 项目概况提前输出机会点；
- 项目概况提前输出大单品候选；
- 环境分析阶段直接输出客户最终机会点；
- 机会洞察前提前输出战略承接；
- 用确认点、用户反馈或 Skill 规则作为客户版页面主要资料依据；
- 页面正文写成长篇报告；
- 金句页写多句解释；
- 反问页写多句解释；
- 最终生成 Word 时默认调用 soffice。
