# 具体机会点洞察执行说明

## 1. 本阶段目的

本阶段的目的，是基于研究分析层的证据，推导出 2-3 个具体机会点。

机会点必须具体到消费者能感知、能表达、能传播的词或方向。

本阶段不是输出品牌定位，也不是直接输出大单品结论，更不是写传播口号。

## 2. 输入依据

本阶段只能基于以下内容进行判断：

1. 行业趋势分析；
2. 消费者需求 / 心智分析；
3. 竞品格局分析；
4. 企业自身分析；
5. 客户资料与品牌资料中可验证的信息。

不得把以下内容作为机会点依据：

- MEMORY.md；
- agent_memory；
- 项目执行记录；
- 用户反馈；
- Skill 规则；
- AGENTS.md；
- 未经验证的主观判断；
- 为了让结论好看而编造的资料。

## 3. 核心推导公式

机会点必须通过四维交叉推导：

行业趋势 + 消费者需求/心智 + 竞品格局 + 企业自身优势 = 具体机会点

四维含义：

1. 行业趋势：
这个方向是否符合行业变化、品类变化、政策变化、渠道变化或消费变化。

2. 消费者需求/心智：
消费者是否真的在意，是否能用自己的语言表达，是否存在明显痛点或未满足需求。

3. 竞品格局：
竞品是否已经占住这个方向。如果竞品已经强势占据，则不能轻易作为客户机会点。

4. 企业自身优势：
客户是否有产品、技术、供应链、渠道、内容、专家、品牌资产等能力支撑这个方向。

## 4. 机会点具体性要求

机会点必须是具体词、具体状态、具体利益、具体场景或具体方向。

好的机会点应接近：

- 软
- 0 添加
- 不垮气色
- 不刺激
- 快速止痒
- 夜间修护
- 熬夜修护
- 饭后舒服
- 孩子愿意吃
- 出门不尴尬

不好的机会点包括：

- 做差异化
- 做健康
- 做高端
- 提升品牌力
- 有效食养
- 功能场景
- 可信赖
- 专业化
- 日常化功能食品
- 高感知功能场景
- 制药级药食同源
- 有验证依据
- 安全有效
- 长期调理
- 品牌升级
- 科研背书

这些词可以作为背景、趋势、定位素材或支撑理由，但不能直接作为最终机会点。

## 5. 机会点压缩规则

如果候选机会点过泛，必须继续压缩。

压缩路径：

企业语言 → 消费者语言
抽象利益 → 具体感知
行业概念 → 品类表达
功能描述 → 使用结果
内部优势 → 购买理由

示例：

“制药级药食同源”不能直接作为机会点
需要继续压缩为：消费者到底感知到什么？例如“吃得安心”“气色不垮”“饭后舒服”等。

“功能场景”不能直接作为机会点
需要继续压缩为：具体是什么场景？消费者在这个场景里要解决什么问题？

## 6. 机会点具体性检查表

每个候选机会点都必须经过以下检查：

| 检查项 | 判断 |
|---|---|
| 机会点名称 | 具体词或方向 |
| 消费者是否能感知 | 是 / 否 |
| 消费者是否能表达 | 是 / 否 |
| 消费者为什么在意 | 具体原因 |
| 竞品是否已经占住 | 是 / 否 / 资料不足 |
| 企业自身是否能支撑 | 是 / 否 / 资料不足 |
| 是否能被产品承接 | 是 / 否 |
| 是否能被品牌传播 | 是 / 否 |
| 是否还需要继续压缩 | 是 / 否 |
| 阶段判断 | 保留 / 调整 / 淘汰 |

## 7. 最终输出格式

确认点 2 输出时，应包含：

一、四维交叉分析表

| 维度 | 关键发现 | 对机会点的启发 | 资料依据 |
|---|---|---|---|

二、候选机会点清单

| 候选机会点 | 来源证据 | 初步判断 |
|---|---|---|

三、机会点具体性检查

使用第 6 节检查表逐个检查。

四、最终建议保留机会点

最终不超过 2-3 个。

每个机会点必须写清楚：

1. 机会点名称；
2. 消费者为什么在意；
3. 竞品为什么没有占住；
4. 企业为什么能支撑；
5. 后续可由什么产品 / 场景 / 传播承接。

## 8. 本阶段禁止事项

禁止：

- 把泛词直接写成机会点；
- 把企业优势直接写成机会点；
- 把品牌定位直接写成机会点；
- 把传播口号直接写成机会点；
- 没有四维证据就输出机会点；
- 为了迎合用户反馈强行保留机会点；
- 机会点超过 3 个；
- 在本阶段直接指定战略大单品；
- 在本阶段直接生成完整战略承接方案；
- 引用 MEMORY.md、agent_memory、确认点、用户反馈、Skill 规则作为机会点依据。
