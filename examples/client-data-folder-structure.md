# 客户资料文件夹结构说明

本仓库不会上传 inputs/ 客户资料文件夹。

原因：
客户资料、品牌手册、产品手册、销售数据、PDF、Word、PPT 等都属于项目资料或敏感资料，不应上传到 GitHub。

团队成员下载本 Skill 后，需要在本地项目根目录手动新建：

inputs/客户资料/客户名称/

示例：

inputs/
└── 客户资料/
    └── 御棠源/
        ├── 客户基础信息.md
        ├── 公司介绍.docx
        ├── 品牌手册.pdf
        ├── 产品手册.pdf
        ├── 竞品名单.md
        └── 消费者调研资料.md

启动 Skill 时输入：

请按照本项目 Skill 执行。本次客户资料在 inputs/客户资料/客户名称/ 中，请从确认点 1 开始。

注意：
- 每个客户单独建一个文件夹；
- 不要把多个客户资料混在一起；
- 不要把 inputs/ 上传到 GitHub；
- inputs/ 已被 .gitignore 忽略；
- 客户资料只保存在本地或公司内部安全空间。
