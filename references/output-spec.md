# 品牌营销战略建议书逐页 PPT 脚本输出规范

## 1. 文件目的

本文件用于约束“品牌营销战略建议书逐页 PPT 内容脚本”的输出质量，确保生成内容像 PPT 页面脚本，而不是长篇报告。

本文件同时用于约束章节边界，确保每个章节只完成本章节任务，不提前释放后文战略结论，避免在项目概况、问题诊断、环境分析等前置章节中提前输出机会点、战略承接或大单品判断。

## 2. 逐页脚本的基本格式

每一页必须包含：

- 页码
- 页面标题
- 页面类型
- 页面正文
- 资料依据

标准格式如下：

```markdown
第 X 页：页面标题

页面类型：
目录页 / 章节页 / 结论页 / 分析页 / 数据页 / 对比页 / 表格页 / 金句页 / 反问页 / 路径页 / 总结页

页面正文：
……

资料依据：
……
```

## 3. 页面标题规则

页面标题必须是观点型标题，而不是普通说明型标题。

好的标题示例：

- 当前问题不是产品太多，而是缺少清晰的增长主线
- 消费者真正能感知的是具体状态，而不是企业资质
- 竞品普遍在讲专业，但没有形成一个可传播的消费词

不好的标题示例：

- 客户情况介绍
- 产品分析
- 竞品分析
- 机会点介绍

## 4. 页面正文规则

页面正文必须短、清楚、结构化，适合转成 PPT 页面。

要求：

- 每页只讲一个核心观点；
- 正文优先使用 2-4 条结构化要点；
- 不写长段报告；
- 不写小作文；
- 不在一页里塞入多个复杂判断；
- 每页正文原则上不超过 150-250 字，必要数据页可适度增加。

## 5. 不同页面类型的写法

### 目录页

只展示客户可见一级目录，不展开解释。

### 章节页

展示完整目录，并突出当前章节。不要写大量正文。

### 结论页

包含一个核心结论和 2-3 条支撑理由。

### 分析页

包含一句观点判断、2-4 条证据、必要时一行阶段判断。

### 数据页

包含一个数据结论、关键数据点或表格说明，以及一句“这说明什么”。

### 对比页

用于竞品、渠道、产品或消费者差异对比。必须有对比维度和阶段结论。

### 表格页

表格必须服务一个明确结论，不得只是堆资料。

### 金句页

只保留一句话，不写解释。

### 反问页

只保留一个问句，不写解释。

### 路径页

用于表达流程、推导链条、战略承接路径或推进步骤。

## 6. 章节边界规则

逐页内容必须按客户可见一级目录分批输出。

每一批开始前必须先输出：

```markdown
本批生成内容：第 X 部分「章节名」

本章任务：
1. ……
2. ……
3. ……

本章不回答：
1. ……
2. ……
3. ……

本章预计页面：
第 X 页：……
第 X+1 页：……
```

然后再开始逐页生成。

## 7. 项目概况章节边界

项目概况只回答：

- 本次项目背景是什么；
- 客户为什么需要做这份建议书；
- 客户提出了哪些表层诉求；
- 本项目的研究路径是什么；
- 本报告将如何展开。

项目概况不得输出：

- 具体机会点名称；
- 战略大单品候选；
- 产品承接结论；
- 最终战略方向；
- 后续打法；
- “本建议书将以 XXX 为核心机会点”；
- “后续将优先判断 XXX 作为战略大单品”。

## 8. 问题诊断章节边界

问题诊断只回答：

- 客户表层问题是什么；
- 零散问题如何归因；
- 表面问题和根本问题是什么；
- 为什么这些问题本质上指向战略方向不清晰、增长主线不明确。

问题诊断不得输出：

- 具体机会点；
- 具体产品承接；
- 最终战略方案；
- 大单品候选。

## 9. 环境分析章节边界

环境分析只回答：

- 行业如何变化；
- 消费者需求 / 心智如何变化；
- 竞品格局是什么；
- 企业自身资产和问题是什么。

环境分析不得输出：

- 客户最终机会点；
- 客户应该占据什么；
- 大单品候选；
- 战略承接结论。

## 10. 机会洞察章节边界

机会洞察章节正式输出：

- 四维交叉分析；
- 具体机会点；
- 机会点具体性检查；
- 最终机会点清单。

具体机会点只能在本章节正式提出。

## 11. 战略承接章节边界

战略承接必须建立在机会点已经正式输出之后。

本章节回答：

- 哪个机会点作为主线；
- 现有产品池如何承接；
- 如果涉及大单品，哪个产品可作为候选；
- 产品、渠道、传播如何围绕机会点形成合力；
- 后续推进节奏是什么。

## 12. 资料依据规则

逐页脚本中的资料依据应优先引用：

1. 客户资料；
2. 品牌手册；
3. 官网 / 官方资料；
4. 行业报告；
5. 政策文件；
6. 消费者调研资料；
7. 竞品官网 / 旗舰店 / 公开资料；
8. 研究分析层中的资料来源。

不得把以下内容作为客户版页面主要资料依据：

- 确认点 1；
- 确认点 2；
- 确认点 3；
- 用户反馈；
- Skill 规则；
- AGENTS.md；
- MEMORY.md；
- agent_memory；
- 项目执行记录。

如果需要引用前序内部判断，应转译为：

“依据前序研究分析。”

但不能把确认点或用户反馈作为客户可见资料来源。

## 13. 信息释放顺序

逐页稿必须按证据和结论逐步释放信息：

客户现状与问题
→ 行业环境
→ 消费者证据
→ 竞品证据
→ 企业自身证据
→ 机会点
→ 战略承接
→ 推进节奏

不得在项目概况、问题诊断、环境分析阶段提前亮出机会点或战略承接答案。

## 14. 禁止事项

逐页脚本中禁止：

- 把项目概况写成战略结论预告；
- 在项目概况提前输出机会点；
- 在项目概况提前输出大单品候选；
- 在环境分析阶段直接输出客户最终机会点；
- 在机会洞察前提前输出战略承接；
- 用确认点、用户反馈或 Skill 规则作为客户版页面主要资料依据；
- 把页面正文写成长篇报告；
- 一页塞入多个复杂判断；
- 金句页写多句解释；
- 反问页写多句解释；
- 跨章节提前释放后文结论。
