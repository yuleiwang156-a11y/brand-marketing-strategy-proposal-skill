# 品牌营销战略建议书逐页 PPT 脚本输出规范

## 1. 文件目的

本文件用于约束“品牌营销战略建议书逐页 PPT 内容脚本”的输出质量，确保生成内容像 PPT 页面脚本，而不是长篇报告。

本文件同时用于约束章节边界，确保每个章节只完成本章节任务，不提前释放后文战略结论，避免在项目概况、问题诊断、环境分析等前置章节中提前输出机会点、战略承接或大单品判断。

## 2. 逐页脚本的基本格式

过程审核版每一页必须包含：

- 页码
- 页面标题
- 页面类型
- 页面正文
- 资料依据

过程审核版标准格式如下：

```markdown
第 X 页：页面标题

页面类型：
目录页 / 章节页 / 结论页 / 分析页 / 数据页 / 对比页 / 表格页 / 金句页 / 反问页 / 路径页 / 总结页

页面正文：
……

资料依据：
……
```

过程审核版用于对话中分批展示，方便用户审核 Agent 是否按规则执行。它不是最终 Word / PPT 制作版格式。

## 2.1 最终 Word 文档定义

最终 Word 文档不是过程记录，也不是 Agent 执行日志。

最终 Word 文档等同于“给 PPT 制作方使用的页面脚本”。最终 Word / PPT 制作版中每页只保留：

1. 页码；
2. 页面标题；
3. 页面正文内容。

最终 Word 中不得出现：

- 页面类型；
- “页面正文：”标签；
- 资料依据；
- “资料依据：”标签；
- “阶段判断：”固定标签；
- Agent 过程说明；
- 内部审核说明；
- Skill 执行说明；
- 质量检查报告；
- 给用户看的操作提示。

最终 Word 不再使用以下过程审核版格式：

```markdown
第 X 页：页面标题

页面类型：
……

页面正文：
……

资料依据：
……
```

最终 Word 应使用以下 PPT 制作版格式：

```markdown
第 X 页：页面标题

页面正文内容……
```

这里的“页面正文内容”不写“页面正文：”四个字，只直接呈现未来 PPT 页面上要出现的内容。

以下内容属于“生成过程控制信息”，可以在对话中出现用于用户审核，但不得写入最终 Word 文档：

- 本批生成内容；
- 本部分对应客户可见一级目录；
- 本章任务；
- 本章不回答；
- 本章预计页面；
- 本部分到此暂停；
- 请确认是否通过；
- 通过后可回复；
- 当前阶段说明；
- Agent 执行说明；
- 质量检查报告；
- 总体检查结论；
- 是否通过 Skill 规则检查；
- 任何给用户看的操作提示。

质量检查报告只能在对话中单独输出，或另存为独立文件（例如 `outputs/质量检查报告.md`），不得写入最终 Word 文档末尾。最终 Word 的最后一页应是客户可见的总结页、下一步建议页或战略推进页。

资料依据仍可在内部审核阶段和过程审核版中使用；如需保留资料依据，必须单独生成内部文件，例如 `outputs/资料依据清单.md`。不得把资料依据写入客户可见 Word。

## 3. 页面标题规则

页面标题必须是观点型标题，而不是普通说明型标题。

好的标题示例：

- 当前问题不是产品太多，而是缺少清晰的增长主线
- 消费者真正能感知的是具体状态，而不是企业资质
- 竞品普遍在讲专业，但没有形成一个可传播的消费词

不好的标题示例：

- 客户情况介绍
- 产品分析
- 竞品分析
- 机会点介绍

## 4. 页面正文规则

页面正文必须短、清楚、结构化，适合转成 PPT 页面。

要求：

- 每页只讲一个核心观点；
- 正文优先使用 2-4 条结构化要点；
- 不写长段报告；
- 不写小作文；
- 不在一页里塞入多个复杂判断；
- 每页正文原则上不超过 150-250 字，必要数据页可适度增加。

## 5. 不同页面类型的写法

### 目录页

只展示客户可见一级目录，不展开解释。

### 章节页

展示完整目录，并突出当前章节。不要写大量正文。

### 结论页

包含一个核心结论和 2-3 条支撑理由。

### 分析页

包含一句观点判断和 2-4 条证据。阶段判断只作为内部收束思路，不在最终 Word 中以“阶段判断：”固定标签出现。

### 数据页

包含一个数据结论、关键数据点或表格说明，以及一句“这说明什么”。

### 对比页

用于竞品、渠道、产品或消费者差异对比。必须有对比维度和阶段结论。

### 表格页

表格必须服务一个明确结论，不得只是堆资料。

### 金句页

只保留一句话，不写解释。

### 反问页

只保留一个问句，不写解释。

### 路径页

用于表达流程、推导链条、战略承接路径或推进步骤。

## 6. 章节边界规则

逐页内容必须按客户可见一级目录分批输出。

每一批开始前必须先在对话中输出以下过程控制信息，用于用户审核，不属于最终 Word 内容：

```markdown
本批生成内容：第 X 部分「章节名」

本章任务：
1. ……
2. ……
3. ……

本章不回答：
1. ……
2. ……
3. ……

本章预计页面：
第 X 页：……
第 X+1 页：……
```

然后再开始逐页生成。

生成最终 Word 时，必须删除上述过程控制信息，只保留客户可见页面脚本。

## 6.1 输出模式区分

### 过程审核版

在对话中分批生成时，可以保留：

- 页面类型；
- 资料依据；
- 本章任务；
- 本章不回答；
- 本章预计页面。

用途：方便用户审核 Agent 是否按规则执行。

### 最终 Word / PPT 制作版

最终生成 Word 时必须删除：

- 页面类型；
- 资料依据；
- 页面正文标签；
- 本章任务；
- 本章不回答；
- 本章预计页面；
- 质量检查报告；
- 内部过程说明。

用途：作为真正交给 PPT 制作方的页面脚本。

## 7. 项目概况章节边界

项目概况只回答：

- 本次项目背景是什么；
- 客户为什么需要做这份建议书；
- 客户提出了哪些表层诉求；
- 本项目的研究路径是什么；
- 本报告将如何展开。

项目概况不得输出：

- 具体机会点名称；
- 战略大单品候选；
- 产品承接结论；
- 最终战略方向；
- 后续打法；
- “本建议书将以 XXX 为核心机会点”；
- “后续将优先判断 XXX 作为战略大单品”。

## 8. 问题诊断章节边界

问题诊断只回答：

- 客户表层问题是什么；
- 零散问题如何归因；
- 表面问题和根本问题是什么；
- 为什么这些问题本质上指向战略方向不清晰、增长主线不明确。

问题诊断不得输出：

- 具体机会点；
- 具体产品承接；
- 最终战略方案；
- 大单品候选。

## 9. 环境分析章节边界

环境分析只回答：

- 行业如何变化；
- 消费者需求 / 心智如何变化；
- 竞品格局是什么；
- 企业自身资产和问题是什么。

环境分析不得输出：

- 客户最终机会点；
- 客户应该占据什么；
- 大单品候选；
- 战略承接结论。

## 10. 机会洞察章节边界

机会洞察章节正式输出：

- 四维交叉分析；
- 具体机会点；
- 机会点具体性检查；
- 最终机会点清单。

具体机会点只能在本章节正式提出。

## 11. 战略承接章节边界

战略承接必须建立在机会点已经正式输出之后。

本章节回答：

- 哪个机会点作为主线；
- 现有产品池如何承接；
- 如果涉及大单品，哪个产品可作为候选；
- 产品、渠道、传播如何围绕机会点形成合力；
- 后续推进节奏是什么。

## 12. 资料依据规则

逐页脚本中的资料依据应优先引用：

1. 客户资料；
2. 品牌手册；
3. 官网 / 官方资料；
4. 行业报告；
5. 政策文件；
6. 消费者调研资料；
7. 竞品官网 / 旗舰店 / 公开资料；
8. 研究分析层中的资料来源。

不得把以下内容作为客户版页面主要资料依据：

- 确认点 1；
- 确认点 2；
- 确认点 3；
- 用户反馈；
- Skill 规则；
- AGENTS.md；
- MEMORY.md；
- agent_memory；
- 项目执行记录。

如果需要引用前序内部判断，应转译为：

“依据前序研究分析。”

但不能把确认点或用户反馈作为客户可见资料来源。

## 13. 信息释放顺序

逐页稿必须按证据和结论逐步释放信息：

客户现状与问题
→ 行业环境
→ 消费者证据
→ 竞品证据
→ 企业自身证据
→ 机会点
→ 战略承接
→ 推进节奏

不得在项目概况、问题诊断、环境分析阶段提前亮出机会点或战略承接答案。

## 13.1 页间逻辑承接规则

逐页内容不能只是一页一页孤立生成。每一页都必须形成自然的汇报推进：

上一页结论
→ 本页进一步拆解 / 验证 / 推进
→ 下一页需要继续回答的新问题

每生成一页时，Agent 需要在内部完成“页间逻辑检查”：

- 上一页讲了什么；
- 本页承接了什么；
- 本页推进了什么；
- 下一页为什么应该出现。

这些检查只用于内部生成，不得在最终 Word 中机械写出“承接上一页 / 引出下一页”等说明。最终页面只呈现自然的汇报逻辑。

页面之间不得出现：

- 每页像独立笔记；
- 上下页之间跳跃；
- 前一页刚讲问题，后一页突然给结论；
- 没有铺垫就进入机会点；
- 没有机会点确认就进入产品承接。

每页应是汇报故事线中的一个镜头，并形成：

问题出现
→ 问题被拆开
→ 外部环境验证
→ 竞品格局对照
→ 企业自身回看
→ 机会点浮现
→ 战略承接落地

页面正文应在内部回答：这一页为什么现在出现，它承接了上一页什么问题，它把听众带向下一页什么判断。但不得在最终 Word 中机械写“承接上一页 / 引出下一页”。

## 13.2 客户可见第一视角表达规则

最终 Word 是给客户汇报的页面脚本，不是内部分析日志。客户版汇报中，“客户”应尽量替换为具体品牌名、企业名或“品牌自身”。

不得使用以下内部口吻：

- 客户当前的问题是……
- 客户资料显示……
- 客户希望……
- 客户当前的核心诉求……
- 该客户……
- 本客户……

应改为客户可见表达：

- 御棠源当前的问题是……
- 从御棠源现有资料看……
- 御棠源希望解决的是……
- 当前品牌增长的核心矛盾是……
- 对御棠源而言……
- 本项目需要回答的是……

如果是其他品牌，自动替换为对应品牌名称、企业名称或“品牌自身”。

## 13.3 最终 Word 生成前清洗规则

生成 Word 文档前，必须做一次“PPT制作稿清洗”。

必须删除：

1. 页面类型；
2. 页面正文标签；
3. 资料依据；
4. 资料依据标签；
5. 阶段判断标签；
6. 本批生成内容；
7. 本章任务；
8. 本章不回答；
9. 本章预计页面；
10. 暂停确认语；
11. 用户操作提示；
12. 质量检查报告；
13. Agent 过程说明；
14. Skill 执行过程说明；
15. 任何内部审核说明。

必须替换：

1. “客户” → 具体品牌名 / 企业名 / 品牌自身；
2. “客户资料显示” → “从现有资料看”或“从具体品牌现有资料看”；
3. “客户当前问题” → “具体品牌当前问题”；
4. “客户版资料依据”等内部表达不得进入正文。

必须优化：

1. 是否大量重复“一句话 + 编号列表 + 阶段判断”；
2. 是否有过多“资料显示 / 从现有资料看 / 本报告将 / 本建议书不会 / 后续需要 / 该页面说明 / 本章形成 / 当前资料中出现”等内部分析口吻；
3. 是否缺少过渡页、金句页、反问页、方向页或总结页；
4. 是否每页都能自然承上启下；
5. 是否更像 PPT 页面，而不是研究报告。

最终交付应分开：

1. 客户可见 Word / PPT 制作版：只包含页码、页面标题和正文内容；
2. 内部质量检查：在对话中输出，或另存为独立 Markdown 文件。

## 13.4 PPT 上屏文案与反格式化表达规则

最终 Word 的正文要面向 PPT 制作，语言应更短、更直接、更有画面感。

避免以下表达：

- 资料显示……
- 从现有资料看……
- 本报告的核心任务不是……而是……
- 本建议书不会直接……而是……
- 阶段判断：……
- 后续需要……
- 该页面说明……
- 本章形成……
- 当前资料中出现……

应改为更直接的汇报表达。例如：

- “本次要回答四个核心问题：增长问题出在哪里？市场机会在哪里？御棠源应该抓住什么？产品、渠道、传播如何形成合力？”
- “御棠源的产品已经不止一类：海棠果汁、海棠复合果汁、NFC 系列、海棠果干，共同构成八棱海棠产品池。”
- “真正要被记住的，不是企业有多少优势，而是消费者为什么选择这一瓶。”

最终 Word 页面正文不得大量重复以下结构：

```text
XXX。
1. XXX；
2. XXX；
3. XXX；
4. XXX。
阶段判断：XXX。
```

同一章节内，连续页面不得都使用“一句话 + 编号列表 + 阶段判断”的格式。如果连续 3 页出现类似结构，必须主动改写页面表达方式。

“阶段判断”可以作为内部思考，但不得在最终 Word 中作为固定标签出现。处理方式：

1. 如果阶段判断很重要，可以升级为页面标题；
2. 如果适合作为转折，可以单独做成金句页、方向页或转折页；
3. 如果只是收束语，可以自然写在正文最后一句；
4. 不要机械写“阶段判断：”。

## 13.5 页面表达多样性规则

最终 Word 不应几乎全部由分析页、结论页、路径页构成。每一章应根据内容需要穿插不同页面表达方式，包括：

目录页、章节页、反问页、金句页、分析页、数据页、对比页、表格页、路径页、总结页、转折页、关键判断页。

要求：

1. 连续 4-5 页密集分析后，应考虑加入一页过渡页、金句页、反问页、方向页或总结页；
2. 从“问题诊断”进入“竞争环境”前，应有承接页；
3. 从“竞争环境”进入“机会洞察”前，应有转折页；
4. 从“机会洞察”进入“战略承接”前，应有承接页；
5. 不要让整份建议书像分析笔记。

参考样板报告时，只学习页面表达逻辑：一页解决一个判断，数据页用数据支撑结论，竞品页先展示事实再收束判断，机会页集中收束前文证据，关键转折处使用目录页、章节页、方向页或总结页。不得复制样板报告的数据、行业内容、客户内容、品牌结论和页面文案。

## 14. 禁止事项

逐页脚本中禁止：

- 把项目概况写成战略结论预告；
- 在项目概况提前输出机会点；
- 在项目概况提前输出大单品候选；
- 在环境分析阶段直接输出客户最终机会点；
- 在机会洞察前提前输出战略承接；
- 用确认点、用户反馈或 Skill 规则作为客户版页面主要资料依据；
- 把页面正文写成长篇报告；
- 一页塞入多个复杂判断；
- 金句页写多句解释；
- 反问页写多句解释；
- 跨章节提前释放后文结论；
- 把本章任务、本章不回答、本章预计页面等过程控制信息写入最终 Word；
- 把质量检查报告写入最终 Word；
- 把页面类型、页面正文标签、资料依据、资料依据标签写入最终 Word；
- 在最终 Word 中写“阶段判断：”固定标签；
