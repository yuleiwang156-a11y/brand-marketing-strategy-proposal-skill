# 品牌营销战略建议书生成 Skill：系统总览

## 1. 本 Skill 是什么

本 Skill 是一个用于生成“品牌营销战略建议书”的交付型 Skill，核心目标是帮助团队稳定产出可交付的 Word 版 PPT 逐页内容脚本。

它不是通用品牌全案生成器，也不是品牌百科工具。它不负责穷尽所有品牌理论、品牌资产模型或营销工具，而是围绕“从客户资料到建议书初稿”的交付过程，建立稳定的判断顺序、输出格式和质量边界。

## 2. 本 Skill 解决什么问题

本 Skill 解决的是：团队在接到客户资料后，如何稳定完成从理解客户到生成建议书初稿的完整工作链条。

它重点规范以下问题：

- 如何判断客户资料是否足够支撑建议书生成；
- 如何完成客户需求理解与项目问题定义；
- 如何开展行业、消费者、竞品、企业自身四类研究分析；
- 如何从研究分析中推导具体机会点；
- 如何搭建客户可见一级目录与内部分析结构；
- 如何逐页生成像 PPT 的内容脚本；
- 如何在输出前完成建议书质量检查。

## 3. 适用场景

本 Skill 适用于以下场景：

- 客户需要品牌营销战略建议书；
- 客户诉求包括增长、定位、大单品、品牌升级、品类选择、产品结构、传播主线等；
- 最终交付物是 Word 版 PPT 逐页内容脚本；
- 需要输出 50-60 页左右的建议书初稿；
- 团队需要在不同项目中保持相对稳定的分析路径、确认节点和页面表达质量。

## 4. 不适用场景

本 Skill 不适用于以下场景：

- 纯招商方案；
- 纯年度传播方案；
- 纯视觉设计方案；
- 纯品牌命名方案；
- 纯电商运营方案；
- 与品牌营销战略无关的执行类任务。

如果客户需求主要是执行排期、物料制作、广告投放、渠道招商、视觉设计或电商运营，本 Skill 只能作为上游战略判断参考，不应直接作为完整交付框架使用。

## 5. 核心流程

本 Skill 的核心流程链条如下：

客户资料
→ 资料完整性判断
→ 确认点 1：客户需求理解与项目问题定义
→ 研究分析层：行业、消费者、竞品、企业自身
→ 确认点 2：具体机会点洞察
→ 确认点 3：客户可见一级目录与内部分析结构
→ 按客户可见一级目录逐部分生成 PPT 页面脚本
→ 质量检查
→ 生成 Word 文档

其中，确认点 1、确认点 2、确认点 3 是必须暂停等待用户确认的关键节点，不得跳过。

## 6. 核心判断逻辑

本 Skill 的核心判断逻辑是：

行业趋势 + 消费者需求/心智 + 竞品格局 + 企业自身优势 = 具体机会点

机会点不能只来自单一维度，也不能只凭客户诉求、竞品动作或内部产品优势直接得出。它必须经过四维交叉推导，并回答消费者为什么在意、竞品是否没有占住、企业自身是否能支撑、能否被产品或品牌表达承接。

机会点必须具体到消费者能感知、能表达、能传播的词或方向。不能停留在“专业化、可信赖、有效、健康、高端、功能场景”等泛词，也不能把“提升品牌力”“做差异化”“做大单品”直接当作机会点。

## 7. 本 Skill 的输出原则

本 Skill 的输出必须遵循以下原则：

- 不编造数据；
- 不跳过确认点；
- 不直接套模板；
- 不提前释放后文结论；
- 不把项目概况写成战略结论预告；
- 不把确认点、用户反馈、Skill 规则作为客户版页面资料依据；
- 逐页脚本要像 PPT，不要像长报告；
- 最终输出按客户可见一级目录分批进行。

逐页脚本应使用 PPT 化表达：页面标题要有观点，页面正文要短、结构化、适合上屏。分析页应呈现观点判断与证据，结论页应突出核心结论与支撑依据，金句页和反问页应保持克制，不写成长段说明。

## 8. 与通用品牌全案 Skill 的区别

本 Skill 不追求覆盖所有品牌战略工具，也不默认输出使命、愿景、价值观、品牌人格、视觉体系、广告语等完整品牌全案内容。

通用品牌全案 Skill 通常更关注品牌体系的完整建设，而本 Skill 更关注“建议书交付稳定性”和“战略机会点推导质量”。它的重点不是把所有品牌模块都写全，而是帮助团队基于客户资料形成清晰的问题定义、研究分析、机会判断、目录结构和逐页表达。

因此，只有当使命、愿景、价值观、品牌人格、视觉体系、广告语等内容与本次战略机会判断和建议书交付直接相关时，才应作为必要内容进入建议书，而不是默认生成。

## 9. 样板报告的使用原则

样板报告只用于学习：

- 汇报结构；
- 信息释放顺序；
- 页面表达方式；
- 证据到结论的推导方式。

不得复制样板报告的具体行业数据、客户内容、品牌结论、页面文案或图表。

样板报告的价值在于帮助 Agent 理解一份成熟建议书如何组织材料、铺垫问题、释放证据、形成判断和推进结论，而不是作为可直接套用的内容模板。不同客户、行业、品类和资料条件下，必须重新基于本次资料完成判断。
